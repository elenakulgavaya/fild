fild
====

|downloads| |python| |license| |ci|

FILD is a lightweight Python library for automated contract testing.
FILD allows you to store a descriptive structure of the API and use it
for generating test data based on Faker.

The basic type for usage is ``Field``. Any other types should inherit from it.

``Field`` supports the description of the API parameter:

- ``name`` – key in JSON
- ``required`` – identifies whether the key must be present
- ``allow_none`` – value can be ``null``
- ``default`` – default value or callable generator

Abstract method ``generate_value`` specifies the rules used to generate
values. When defining a new type inherited from ``Field``, this method
should be overridden to describe the generation procedure.

The library supports multi-level hierarchies with Arrays and Dictionaries.

Dictionary utilities:

- ``filter_dict`` – apply value/callable filters
- ``merge_with_updates`` – merge dictionaries with overrides or additions
- ``normalize`` – sort nested lists for deterministic comparison


Examples
--------

Field objects are used to describe data structures:

.. code-block:: python

   from fild.sdk import Dictionary, Enum, Int, String, Uuid


   class Gender(Enum):
       Male = 'male'
       Femail = 'female'
       Other = 'other'


   class Customer(Dictionary):
       Id = Uuid(name='id')
       Name = String(name='name', max_len=100)
       Age = Int(name='age', min_val=18, max_val=120)
       Gender = Gender(name='gender', required=False)
       Email = String(name='email')


Based on the description, the data can be generated by calling the
constructor with or without parameters:

.. code-block:: python

   generated_customer = Customer()
   generated_full_customer = Customer(is_full=True)
